# 宠物名 / 物种解析（miloco-miot-pet-register）

从用户一句话里抽两个字段喂 `pet add --name <name> --species <species>`。

## 字段定义
- **`name`（必填）**：宠物的**称呼/名字**（小黑、Max、小橘）。抽不到 → **追问**，不拿品种/物种顶替。
- **`species`（尽量给）**：**猫 / 狗 / 其它**。品种（金毛、英短、柯基…）不是 species——归 `species` 的自然语言或留给外观描述；`species` 优先填大类"猫/狗"。

`name` 与 `species` 独立：species 不确定可先按检出结果（observe 的 `description.species` / `species_guess`）填，绝不用 name 顶替。

## 抽取速查

| 用户原话 | name | species | 备注 |
|---|---|---|---|
| "这是我家猫小黑" | 小黑 | 猫 | |
| "记住这只狗，叫 Max" | Max | 狗 | |
| "我家金毛 Max" | Max | 狗 | 金毛=品种，species 填"狗"；品种可写进外观 |
| "这是小橘"（附图是猫） | 小橘 | 猫 | species 缺 → 用 observe 检出结果补 |
| "我家那条狗"（无名） | **（空，追问）** | 狗 | 追问："这只狗叫什么名字？" |
| "帮我登记这两只猫"（多只） | **（空，追问 + 拆）** | 猫 | 一次只登记一只；先问主体那只叫什么，其它分别再传 |

## 边界
- **只给品种没给名**（"这是只英短"）→ 追问名字（英短不是名字）。
- **一句给多只**（"小黑和小白"）→ 一次只注册一只；先确认本次注册哪只 + 其素材，另一只完成后再来一次（observe 也只处理主体那只，见 SKILL `multiple_pets`）。
- **名字含空格/emoji/表情** → 原样传 `--name`（后端按字符串存），不自行改写。
- **重名**：`pet add` 前先 `miloco-cli pet list` 查；若同名已存在且用户不是"补充素材"意图 → 提示确认是同一只（补充素材）还是另一只（换名）。
